一、Dom获取&属性操作

1. DOM的相关概念

DOM：Document Object Model 文档对象模型，是用来呈现以及与任意 HTML 或 XML文档交互的API

API 接口的意思,可以理解为方法,函数；

文档: document 一个页面就是一个文档；

元素: 页面上所有的标签都叫元素 element；

节点: 页面上所有的内容都叫节点 node；

DOM 的作用：开发网页内容特效和实现用户交互；

DOM树：将 HTML 文档以树状结构直观的表现出来，我们称之为文档树或 DOM 树；描述网页内容关系的名词；

DOM树的作用：直观的体现了标签与标签之间的关系；

DOM对象：

  • 浏览器根据html标签生成的 JS对象；
  • 所有的标签属性都可以在这个对象上面找到；
  • 修改这个对象的属性会自动映射到标签身上；

document 对象：

  • DOM 里提供的一个对象；
  • 所以它提供的属性和方法都是用来访问和操作网页内容的；
  • 网页所有内容都在document里面；

2. 获取DOM对象

2.1 根据CSS选择器来获取DOM元素

2.2.1 获取匹配的第一个元素

语法： document.querySelector('CSS选择器')

返回值： CSS选择器匹配的第一个元素,一个 HTMLElement对象。 如果没有匹配到，则返回null。

伪数组:有索引, 有length, 但是没有数组的方法,如pop, push等；

2.1 鼠标事件

click 点击事件；

mouseenter 和 mouseleave 鼠标经过和鼠标离开 没有事件冒泡；

mouseover / mouseout 鼠标经过 / 离开

3.事件对象🔥

事件对象 : 当事件发生的时候,和这个事件相关的所有信息,都存在这个对象当中,这个对象就叫事件对象;

环境对象：this 就是环境变量，是函数内部的特殊变量，代表了当前函数运行时所处的环境；每个函数都有this！

粗略判断规则: 谁调用就指向谁（指向可以理解为等于）；

对于普通声明的函数，都会挂载到 window 对象上；

（挂载：作为 window 的一个属性或者方法）

在绑定注册事件时， this 代表的就是 注册事件的元素 ；

5.回调函数

回调函数：callback 作为函数的参数，一开始不执行，回头再调用的函数；

应用场景：

  1. 定时器

2.事件监听

注意：作为参数传入的时候，不可以加括号，加了小括号就表示调用执行；

三、Dom事件进阶

3.1 事件流

3.1.1 事件流🔥

事件流：时间完整执行过程中的流动路劲；描述的是元素在页面接收事件的顺序；

DOM事件流的三个阶段：

  • 捕获：从内到外(从父到子)；
  • 达到目标阶段；
  • 冒泡：从内到外(从子到父)；

3.1.2 事件捕获

事件捕获：document -> html -> body -> father -> son

addEvenListener 有三个参数，第三个参数一般省略，默认为 false ；如果第三个参数传入 true ，表示使用捕获；

target.addEventListener(type, listener, useCapture)

3.1.3 事件冒泡🔥

事件冒泡：事件从最具体的元素（文档树中最深的节点）开始触发，然后向上传播至没有那么具体的元素（文档）(—JS高程)；

事件冒泡： son -> father -> body -> html -> document；

addEventListener第三个参数省略，或者为false；

3.1.4 阻止事件冒泡🔥

e.stopPropagation() 阻止事件冒泡，阻止的是事件的传播，不会阻止函数内代码的运行；

前提, 需要先写上 e 事件对象；

3.1.5 解绑事件

addEventListener 方式，必须使用removeEventListener(事件类型, 事件处理函数, 获取捕获或者冒泡阶段)

匿名函数无法解绑！

3.2 事件委托与阻止默认行为🔥

3.2.1 事件委托

事件委托：把事件委托给父元素，代为处理；

原理：利用事件冒泡，将事件监听绑定到父元素上，通过父元素来监听子元素的事件；

好处：减少事件注册的次数， 提高程序性能；

e.target 代表触发事件的元素；

this 绑定事件的元素；

e.currentTarget 同 this绑定事件的元素；

e.target.tagName 找到合适的标签名；

3.2.2 阻止默认行为

e.preventDefault() 阻止默认行为；

3.3 其他事件

3.3.1 页面加载事件

1.页面加载事件 load
等待页面所有的资源加载完毕,就会去执行回调函数；

2.DOMContentLoaded

当DOM元素加载完成时,就会去执行至回调函数,不包括图片,样式表等；

两者的区别：

  • load 需等待页面所有元素加载完毕，才会去执行回调函数 监听整个页面资源 ，给window加；
  • DOMContentLoaded 当DOM元素执行完毕后，就回去执行回调函数；

5.2.1 延时器 setTimeout

语法 ：setTimeout(回调函数, 延迟时间) 省略了window；

  • window可以省略；
  • 单位 毫秒, 延迟时间,这个参数可以省略, 省略的时候默认为0 ,最小延迟 4ms；

书写方式：

直接写函数

写函数名

5.2.1 setTimeout / setInterval 区别

setTimeout(cb, wait) 只执行一次, 间隔时间到了后, 就会执行；

setInterval(cb, wait) 每隔一段时间, 就会执行一次. 会重复执行, 一直执行. 直到清除定时器；

5.5 本地对象

生命周期：一个对象的创建到清除的过程，数据存在的周期；

5.5.1 本地储存 localStorage

本地储存：localStorage

  • 生命周期永久生效，除非手动删除，否则关闭页面也会一直存在 （记忆）；
  • 可以多窗口（页面）共享（同一浏览器可以贡献：域名一致）（记忆）；
  • 以键值对形式存储 key value；

方法：

1.存储数据 localStorage.setItem(key, value)

2.获取数据 localStorage.getItem(Key)

3.删除本地存储 localStorage.removeItem(key)

4.清空数据 localStorage.clear()

5.5.2 会话储存空间 sessionStorage

  • 生命周期为关闭浏览器窗口；
  • 在同一窗口（页面，Tab下）数据可以共享（同一域名下）；
  • 键值对形式储存；

方法：和localStorage一样

存储数据 sessionStorage.setItem(key, value)

获取数据 sessionStorage.getItem(key)

删除数据 sessionStorage.removeItem(key)

清空数据 sessionStorage.clear()

5.5.3 本地储存复杂数据类型

本地只能存储字符串,无法存储复杂数据类型。需要将复杂数据类型转换成JSON字符串,在存储到本地；

注意：JSON格式：

属性和值，key和value 都有引号，并且统一是双引号（若值是number的话，没有引号）；

1.复杂数据类型（对象，数组） 必须转化为 JSON格式的字符串

JSON.stringify() 将对象转化为 JSON格式的字符串

JSON.parse() JSON格式的字符串转换成对象

对于复杂数据类型，存的时候，要转JSON格式的字符串；取出来，再转化为对象才能使用！

  • JSON.stringify() 将对象转化为JSON格式的字符串；
  • JSON.parse() 将JSON格式的字符串转化为对象；
