初识 ajax

1.客户端与服务器

1.1 客户端与服务器

后台服务端主要职责？

负责 存放和对外提供资源 的电脑；

前台客户端主要职责？

负责 获取和消费资源 的电脑；

1.2 URL 地址的概念&组成

URL地址概念？用于什么场景？

URL（全称是 UniformResourceLocator） 中文叫 统一资源定位符，用于标识互联网上每个资源的唯一存放位置。浏览器只有通过URL地址，才能正确定位资源的存放位置，从而成功访问到对应的资源；

URL由哪几部分组成？

URL地址一般由三部分组成:

客户端与服务器之间的 通信协议；

存有该资源的 服务器名称；

资源在服务器上 具体的存放位置；

1.3 客户端与服务器的通讯过程

分为三个步骤

请求 – 处理 – 响应

1.4 利用浏览器工具分析通讯过程

如何利用浏览器工具来分析通讯过程

  • 打开Chorme浏览器
  • Ctrl + Shift + I（F12） 打开 Chrome 的开发者工具
  • 切换到 Network 面板
  • 选中Doc 页签
  • 刷新页面，分析客户端与服务器的通讯过程

1.5 网页中如何请求数据

如果要在网页中请求服务器上的数据资源，需要用到 **XMLHttpRequest** 对象

XMLHttpRequest（简称 xhr）是浏览器提供的JS成员，通过它，可以请求服务器上的数据资源

最简单的用法 let xhrObj = new XMLHttpRequest()

1.6 数据请求的方式

get 请求：通常用于 获取服务器资源（要资源），例如：根据 URL 地址，从服务器获取 HTML文件、css文件、js文件、图片文件、数据资源等；

post 请求：通常用于 向服务器提交数据（送资源），例如：登录时，向服务器 提交登录信息、注册时向服务器 提交注册信息、添加用户时向服务器 提交用户信息等各种 数据提交操作；

2. 初始 ajax

2.1 ajax 的概念

什么是 Ajax？

Ajax 的全称是 Asynchronous JavaScript And XML（异步 JavaScript 和 xml）；

通俗理解：在网页中利用 XMLHttpRequest 对象和服务器进行数据交互的方式，就是Ajax；

为什么要学习 Ajax？

之前所学的技术，只能把网页做的更美观漂亮，或添加一些动画效果，但还是，Ajax能让我们轻松实现 网页与服务器之间的数据交互；

2.2 jQuery 中的 ajax

jQuery中常用的方法有几个？

  • $.get() get方式的请求，用于获取数据;
  • $.post() post方式的请求，用于提交数据;
  • $.ajax() 比较综合，既可以获取数据，又可以提交数据;

在原生JS中，实现等待页面加载完成后再执行回调函数的代码为 window.addEventListener('load', function () {}) 或者 window.onload(function () {}) ；

jQuery 提供了一个页面加载完成执行函数：$(function () {})
