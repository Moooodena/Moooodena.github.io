函数作用域相关（作用域、垃圾回收机制、闭包、变量提升、函数参数和箭头函数）

1. 作用域

作用域 scope : 就是变量与函数的可访问范围，即作用域控制着变量和函数的可见性和生命周期。

ES6后作用域分为三类：全局作用域，函数作用域和块级作用域，其中函数作用域和块级作用域合称为局部作用域。

1.1 全局作用域

全局作用域 (Global Scope): 中的对象在代码中的任何地方都能访问，其生命周期伴随着页面的生命周期；

直接写在script标签中的js代码，在全局作用域

注意：

  • 为 window 对象动态添加的属性默认也是全局的；
  • 函数中未使用任何关键字声明的变量为全局变量；
  • 尽可能少使用全局变量，防止全局变量被污染！

1.2 局部作用域

局部作用域分为函数作用域和块级作用域（ES6）新增；

1.2.1 函数作用域

在函数内部声明的变量只能在函数内部被访问，外部无法直接访问。

注意：

  • 函数内部声明的变量， 在函数外部无法被访问
  • 函数的形参也是函数的局部变量
  • 不同函数内部声明的变量, 无法相互访问
  • 函数执行完毕后, 函数内部的变量实际被清空了(垃圾回收)
1.2.2 块级作用域

ES6, {} 包裹起来的代码称为代码块, 使用 let 或 const 声明的变量, 在 {} 会产生块级作用域

  • var声明不会产生块级作用域
  • 只有let const 才会产生块级作用域
  • 不同代码块之间的变量无法相互访问
  • 局部作用域声明的变量 外部不能使用

1.3 作用域链

作用域链的本质是底层上的变量查找机制。

  • 在函数被执行时，会优先在当前函数作用域下查找变量；
  • 如果当前作用域下查找不到，会依次逐级查找父级作用域，直到全局作用域；

注意：

  • 嵌套关系的作用域串联起来形成了作用域链；
  • 相同作用域中按着从小到大的规则查找变量；
  • 子作用域能够访问到父级作用域；但父级作用域无法访问子级作用域；

2. 垃圾回收机制

2.1 垃圾回收机制概念

概念：垃圾回收机制(Garbage Collection) 简称 GC

JS中内存的分配和闲置资源的回收都是自动完成的，内存在不使用的时候会被垃圾回收程序自动回收。

以上代码所示，函数 fn 在每一次被调用，执行完毕后，其通过 const 声明的 str 的值都会被垃圾回收器回收，这正是它可以使用 const 声明的原因；

GC的基本思路就是：确定哪个变量不会再使用，然后释放它占用的内存。这个过程是周期性的，即垃圾回收程序每隔一定时间（或者说在代码执行过程中某个预定的收集时间）就会自动运行。

2.2 内存泄漏

不再用到的内存，没有及时释放，就会造成内存泄漏

内存的生命周期：

  • 内存分配：当我们声明变量、函数、对象的时候，系统会自动为他们分配内存
  • 内存使用：即读写内存，也就是使用变量、函数等
  • 内存回收：使用完毕，由垃圾回收自动回收不再使用的内存

注意：

  • 全局变量一般不会回收(关闭页面回收)；
  • 一般情况下局部变量的值, 不用了, 会被自动回收掉

2.3 垃圾回收的策略

现在常见的有两种垃圾回收的方法：

  • 引用计数法（淘汰）
  • 标记清除法
2.3.1 引用计数法 （淘汰）

IE 采用的引用计数方法，定义“内存不再使用” 的标准较为简单，就是看一个对象是否有指向它的引用；

算法：

  1. 跟踪记录每个值被引用的次数；
  2. 如果这个值被引用了一次，那么就记录次数 1 ；
  3. 多次引用的话就会累加；
  4. 如果减少引用次数就会减 1 ；
  5. 如果引用次数为 0 ，则会释放内存；

但是 ！ 引用计数有一个缺陷：循环引用；

如果两个对象相互引用，尽管他们已不再使用，垃圾回收器不会进行回收，导致内存泄露；

2.3.2 标记清除法

标记清除法的核心：

  1. 标记清除法将 “不再使用的对象” 定义为 “无法到达的对象” ；
  2. 从根部（在JS中表示全局变量）出发定时扫描内存中的对象，凡是能从根部到达的对象，都是需要使用的；
  3. 那些无法由根部触发触及到的对象被标记为不再使用，稍后进行回收；

其过程主要分为两个阶段：

  • 标记阶段：把所有活动对象做上标记。
  • 清除阶段：把没有标记（也就是非活动对象）销毁。

标记阶段：

根可以理解成我们的全局作用域，GC从全局作用域的变量，沿作用域逐层往里遍历（对，是深度遍历），当遍历到堆中对象时，说明该对象被引用着，则打上一个标记，继续递归遍历（因为肯定存在堆中对象引用另一个堆中对象），直到遍历到最后一个（最深的一层作用域）节点。

清除阶段：

回收没有打上标记的对象；

闭包 closure

4. 变量提升

4.1 变量提升

变量提升是 JS 中比较奇怪的现象，它允许在声明变量之前即被访问（仅存在于 var 声明的变量）

注意：

  1. 变量在未声明即被访问时会报语法错误；
  2. 变量在 var 声明前即被访问，变量的值为 undefined ；
  3. let / const 声明的变量不存在变量提升；
  4. 变量提升出现在相同的作用域中；
  5. 实际开发中应先声明再访问变量；

4.2 函数提升

定义函数有两种方式：

5. 函数参数

5.1 函数的动态参数 arguments

  • 所有的函数都内置了（默认就存在，自带的）一个 arguments 对象；
  • arguments 是一个伪数组（想想什么是伪数组）， 存储了传递来的所有实参；
  • 它只存在于函数中；

5.2 剩余参数 rest

ES6 引入 rest 参数（形式为...变量名）ES6新增，用于获取函数的多余参数，这样就不需要使用arguments对象了。rest 参数搭配的变量是一个数组，该变量将多余的参数放入数组中。

语法: …变量
rest 剩余参数，是一个真实的数组，存了剩余的实参

5.3 拓展 扩展运算符(…arr)

注意展开运算符与函数的剩余参数的区别：

  • 展开运算符在数组中使用，将数组的元素展开；
  • 剩余参数在函数中使用，它得到一个真数组；
