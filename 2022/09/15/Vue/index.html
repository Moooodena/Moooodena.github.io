如何在App.vue中引入组件？

  1. 通过import ‘引入’ 组件，并为组件起一个自定义名称；

  2. 通过components 属性进行组件 ‘注册’ ；

  3. 在 template 标签内，以’标签形式’进行使用；

在<style></style>中可以设置组件样式，并为了保证设置样式只作用于当前的文件，会设置一个scoped属性；

对于main.js：

1. 插值表达式（声明式渲染） - 插入的某个值

在Vue中通过双花括号的形式来构架插值表达式

语法：{{ 表达式(< > >= == === ! && || 三目运算 变量) }}

简单来说，就是在标签里写js代码，插值表达式就是替换了js中的innerText；

注意：一个vue文件中，声明的变量需要放置在data函数的return对象上！！！

2. MVVM设计模式（面试必考）

设计模式: 是一套被反复使用的、多数人知晓的、经过分类编目的、代码设计经验的总结。

  • MVVM，一种软件架构模式，决定了写代码的思想和层次
    • M： model数据模型 (data里定义)
    • V： view视图 （html页面）
    • VM： ViewModel视图模型 (vue.js源码)
  • MVVM通过数据双向绑定让数据自动地双向同步 不再需要操作DOM
    • V（修改视图） -> M（数据自动同步）
    • M（修改数据） -> V（视图自动同步）

所以！在vue中，不需要再去操作Dom了，而是通过操作数据来实现驱动视图。

面试题：简单描述一下vue的设计模式

答：vue使用的mvvm设计模式。MVVM是Model-View-ViewModel缩写，也就是把MVC中的Controller演变成ViewModel。

Model层代表数据模型，View代表UI组件，ViewModel是View和Model层的桥梁，

数据会绑定到viewModel层并自动将数据渲染到页面中，视图变化的时候会通知viewModel层更新数据 (数据双向绑定)。

3. v-bind - 动态属性

给标签属性设置vue变量的值

  • 语法：v-bind:属性名="vue变量 (写在 data 的return 返回对象中)"
  • 简写：:属性名="vue变量"

关于动态属性的特例： 在使用动态属性渲染图片时，不可以直接将变量的值设置为路径；

原因： v-bind 会把这个路径认为是字符串

解决方案：

  1. 通过 import 的方式引入图片并且赋值给data中的变量；

  2. 通过 require 的方式引入图片并赋值给data中的变量；

以上两种解决方案，你觉得哪一个更好呢？？（面试考点：如何进行代码优化！！！）

答： require 的引入方式更为友好。 当一个vue文件被使用时，那么既然是js代码，那么遵循从上往下依次执行，

import中引入的图片就会在页面渲染初期被加载，而require引入的图片，只有在使用的时候才会被加载，初次开启页面会显得更加的高效；

注意！！！属性值要写在 data 的return 返回对象中！

4. v-on 事件绑定

v-on 绑定事件 替换的是 webapi 中的.onclick, .addEventListener ；

4.1 v-on 事件绑定基本使用

语法: v-on:事件名=”少量代码”

​ v-on:事件名=”methods里函数名”

​ v-on:事件名=”methods里函数名(值)”

       @事件名="methods里函数名"

注意export default{ }里的this，它指的是export default{ }这个对象，data和methods里的属性都挂载在这个对象上；

4.2 v-on的事件对象 —event

v-on 获取事件对象 event

  1. 无参数获取事件对象 我们的事件对象**event默认在第一个形参上**；

  2. 有参数(特指带小括号的方法)，这个小括号会覆盖event默认参数，我们可以用$event的实际参数来代替这个事件对象；

  3. 一般情况下，我们都将事件对象event 放到整个形式参数的最后一位；

4.3 v-on 修饰符

修饰符就是为了简化代码！

语法： @事件名.修饰符=”methods里函数”

​ .stop - 阻止事件冒泡

​ .prevent - 阻止默认行为

​ .once - 程序运行期间, 只触发一次事件处理函数

问题：如果我把修饰符全忘了，影响我开发吗？

答：并不影响，因为修饰符只是在简化我们对事件对象的操作，并不影响事件对象本身；

问题：修饰符与修饰符之间能不能混合使用？

答：可以混合使用，只需要在第一个修饰符后面通过点语法的形式连接后续修饰符即可；

4.4 按键修饰符

例如：.enter .esc

  1. 在登录注册时，输入完账号密码后，敲下回车会立即进行登录操作？

  2. 当输入完某个输入框(账号框)发现输入错误了，点击esc（大部分键盘都在左上角）可以实现清空输入框的操作？

    => 获取输入框的dom元素 => 监听键盘事件(keydown/keyup) 判断keycode是否时我们的目标按键；

5. v-model

5.1 v-model 使用

  1. 数据变化 => 视图自动变化

  2. 视图变化 => 数据自动变化

    语法： v-model="data中定义的变量" 代替的是input中的value属性;

5.2 v-model修饰符

场景：

在表单提交时，需要调用后端的接口，每一个接口所接受的参数都有自己的规则；

1 电话号码必须是数字类型（输入框中所表现的是字符串 .parseFloat()来进行了调整转化）；

2 账号密码不能有前后的空格(输入框前后都可以敲空格 .trim()来去除前后空格)；

常用修饰符：

  1. (常用) .number修饰符： 它可以以parseFloat的方式将输入框中的内容转化为数字；

与parseFloat最大的区别: 如果开头就不是数字，.number会失效，而parseFloat则会转化为NaN；

  1. .trim修饰符： 它可以将输入框中v-model绑定的字符串实时的去除前后空格;

  2. .lazy修饰符： 当在输入框中每输入一个字符时，其实数据都会实时的去变化;这种变化频率在某些情况下显得有一点高，那么可以使用lazy修饰符让数据改变的这一行为推迟到onchange事件时(input框的失焦、input的回车键)

7. v-show和v-if

v-show="变量"

v-if="变量"

变量应该是boolean值，还可以判断当前变量隐式转化为boolean值的状态；

那些数据隐式转化会是false： 0 null undefind '' NaN

v-show 是采用display:none的方式进行显示和隐藏的，其实不管怎么说这个元素还是在dom树上；

v-if 是采用删除、添加dom元素来进行显示和隐藏的， 隐藏时其实是将当前的元素从dom上直接删除；

注意事项： 一般情况下**使用v-show来进行显示隐藏的元素都是静态元素(它内部没有动态的变量控制)**，除此以外均建议使用v-if来进行显示和隐藏；

8. v-for

8.1 v-for基本介绍

语法： v-for="(值, 当前索引) in 需要遍历的数据结构"

和forEach非常相似的；

循环遍历是，需要给v-for标签上添加一个key属性，用来表现当前循环的每一个元素它的唯一 (一般可以用索引来作为它的key)；

口诀： 谁要循环就放在谁上面；

(主要去掌握数组，数组对象的遍历方式，其他的遍历只是vue提供了这样的功能，实际上并不常用)；

8.2 v-for和v-if同时使用

v-for与v-if不允许在同一个标签中使用！！！

8.3 v-for的更新监测

  1. 当v-for遍历的数组发生变化时，(一定是得发生变化) , 会触发v-for的遍历更新；

  2. 如果某个数组处理方式不会改变原数组，那么通过赋值的操作来让数组发生变化；

  3. 如果遍历的数据无法实时响应式，可以使用this.$set()方法来进行强制响应，这一方法其实是应急手段，在迫不得已时（找不到其他解决办法时）再去使用；
